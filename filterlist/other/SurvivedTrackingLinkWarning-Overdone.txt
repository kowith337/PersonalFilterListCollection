!         the issue about seamless navigating that use "XMLHttpRequest" and/or "CORS" to be stuck! so use at your own risk!!

! Soft problematic!
! - You cannot perform certain actions while using its website, so I've decided to add query prune rules for addition, but please
!   note that `removeparam` will clean only first detected parameter (race condition), it cannot clear multiple parameters in
!   each requests, probably some information still slipped through other parameters that cannot removed at that time.
$document,xmlhttprequest,important,domain=facebook.com|messenger.com,removeparam=/^((__)?m_|[a-zA-Z]+?\[\d+?\]|__(c(cr|ft)?|rv|so|tn|xts?)__|logging_|privacy_|ref_?)$/i
$document,xmlhttprequest,important,domain=facebook.com|messenger.com,removeparam=acontext
$document,xmlhttprequest,important,domain=facebook.com|messenger.com,removeparam=rc
$document,xmlhttprequest,important,domain=steamcommunity.com|steamgames.com|steampowered.com,removeparam=/^(curator_|s(e|n)r)$/i
$first-party,document,xmlhttprequest,domain=facebook.com|messenger.com|~mtouch.facebook.com|~touch.facebook.com|~www.facebook.com,removeparam=/^fb_dtsg_?/i
$first-party,important,domain=github.com,removeparam=/^notifications?/i
